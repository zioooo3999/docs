The Tempe Park website architecture consists of three main parts, namely **User**, **Frontend**, **Backend (CMS)**, and **External Services** which are connected according to feature requirements.

---

## 1. User Access to the Website

- The flow begins when the **User** accesses the Tempe Park website via a browser.
- Requests from users will be directed to the **Frontend** built using **Next.js**.
- The frontend is responsible for displaying pages, managing navigation, and handling user interactions.

## 4. Backend (CMS) Responsibilities

Backend has the main responsibilities:

- Manage all website content (articles, events, programs, etc.)
- Providing API for frontend
- Setting user access (admin/editor)

With the use of **Strapi**:

- Non-technical teams can update content without developer assistance.
- Centralized and structured content

---

## 5. Role of Database

The database serves as the primary storage:

- Save all content data
- Supports read/write operations from CMS
- Ensuring data consistency and security
