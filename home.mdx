# Technical Design Document (TDD)

Welcome to the **Technical Design Document (TDD)** for the **Tempe Park** ecosystem. This document serves as the primary technical guide for engineering teams, developers, and stakeholders to understand the system architecture, data flow, and database structure.

---

## 📌 Document Objectives

This document aims to:

- **Standardize Architecture:** Provide a comprehensive overview of the Front-End and Back-End (CMS) system architecture.
- **Developer Onboarding:** Facilitate developer onboarding and maintain consistency across code implementation.
- **Data Schema Reference:** Provide detailed documentation of entity/table definitions and the Entity Relationship Diagram (ERD).

## 🛠️ Tech Stack Summary

The Tempe Park system is built using modern tech stacks to ensure high performance, scalability, and seamless content management:

- **Front-End:** Next.js / React
- **CMS / Back-End:** Strapi Headless CMS
- **Database:** PostgreSQL / MySQL
- **Styling:** Tailwind CSS
