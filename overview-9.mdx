## Entity / Table Definition

This section details the structure of each entity or table used in the Tempe Park database. Each entity represents a primary type of data managed in the system, such as articles, programs, activities, facilities, and other content.

For each entity, information will be provided, including the table name, a description of the table's function, and a list of its attributes or fields, along with their data types, constraints, and explanations. This definition aims to provide a clear picture of the data structure used, thus serving as a reference for the development team during backend implementation and frontend integration.
